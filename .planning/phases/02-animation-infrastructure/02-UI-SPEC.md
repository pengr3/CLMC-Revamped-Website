## Animation Contract

Phase 2's primary output is animation primitives. These are the declared behavioral contracts all primitives must conform to.

### Scroll Reveal — FadeUp

| Property | Value | Source |
|----------|-------|--------|
| Trigger | Element top edge crosses 90% of viewport height (`start: "top 90%"`) | Phase 2 D-04 |
| Enter: opacity | 0 → 1 | — |
| Enter: translateY | 40px → 0px | — |
| Enter duration | 0.7s | discretion — matches Linear.app pacing |
| Enter easing | `power2.out` | discretion — smooth deceleration |
| Exit: opacity | 1 → 0 | Phase 2 D-03 (re-animate each direction) |
| Exit: translateY | 0px → -20px | discretion — subtle upward drift on leave |
| Exit trigger | Element bottom edge leaves viewport top (`end: "bottom top"`) | Phase 2 D-03 |
| Scrub | false — snappy trigger, not scroll-linked | Phase 2 D-04 |
| markers | false in production | — |
| Implementation | GSAP + ScrollTrigger inside `useGSAP()` hook | CLAUDE.md, Phase 2 D-04 |
| Reduced motion | Static: element visible at full opacity, no transform | DSGN-04 |

### Scroll Reveal — Clip/Mask (Hero moments and key visuals)

| Property | Value | Source |
|----------|-------|--------|
| Trigger | Same as FadeUp: `start: "top 90%"` | Phase 2 D-04 |
| Technique | `clip-path: inset(100% 0 0 0)` → `inset(0% 0 0 0)` (bottom-up wipe) | discretion — clean, editorial |
| Duration | 0.9s | discretion — slightly slower than fadeUp for visual weight |
| Easing | `power3.out` | discretion — aggressive deceleration for cinematic snap |
| Exit behavior | Reverse clip on exit (`inset(0% 0 0 0)` → `inset(100% 0 0 0)`) | Phase 2 D-03 |
| Use cases | Hero section images, project gallery reveal, stats/numbers | Phase 2 D-01 |
| Reduced motion | Static: element fully visible, clip-path: none | DSGN-04 |

### Stagger Children

| Property | Value | Source |
|----------|-------|--------|
| Delay between items | 0.08s–0.12s (80–120ms) | Phase 2 D-02 |
| Stagger curve | `power1.out` easing applied to stagger timing | discretion — eased cascade |
| Max simultaneous | 5 children in flight at once (beyond 5: next starts when first completes) | discretion |
| Trigger | Parent container enters viewport, children stagger from that moment | Phase 2 D-04 |
| Applicable to | Grid items, card lists, icon lists, navigation items in mobile overlay | Phase 2 D-02 |
| Reduced motion | Static: all children visible immediately | DSGN-04 |

### Page Transitions

| Property | Value | Source |
|----------|-------|--------|
| Mechanism | Motion `AnimatePresence` wrapping `<main>` content | Phase 2 D-05, CLAUDE.md |
| Enter | opacity: 0 → 1, duration: 300ms | Phase 2 D-05 |
| Exit | opacity: 1 → 0, duration: 300ms | Phase 2 D-05 |
| Easing | `easeInOut` | discretion — symmetrical fade feels deliberate |
| Shell behavior | Navbar and Footer remain mounted, do NOT animate | Phase 2 D-06 |
| Loading state | None — SSG pages load near-instant, no spinner | Phase 2 D-07 |
| Scroll reset | Lenis already handles `scrollTo(0, immediate: true)` on pathname change | LenisProvider.tsx |
| Layout in DOM | `app/layout.tsx`: `<LenisProvider><Navbar /><AnimatePresence><main key={pathname}>{children}</main></AnimatePresence><Footer /></LenisProvider>` | Phase 2 D-06 |
| Reduced motion | AnimatePresence present but duration overridden to 0.01ms by globals.css `prefers-reduced-motion` rule | DSGN-04 |

### Button Hover Micro-Interaction

| Property | Value | Source |
|----------|-------|--------|
| Scale | 1.0 → 1.04x on hover | Phase 2 D-08 (midpoint of 1.03–1.05) |
| Shadow | 0px 0px 0px 0px → 0px 8px 24px rgba(0,0,0,0.5) on hover | discretion — subtle lift |
| Duration | 150ms (matches existing `transition-all duration-150 ease-out` in Button.tsx) | Button.tsx — do not change |
| Implementation | CSS via Tailwind utility classes added to Button.tsx (`hover:scale-[1.04] hover:shadow-[0px_8px_24px_rgba(0,0,0,0.5)]`) | discretion — no Motion needed for this |
| Reduced motion | `prefers-reduced-motion` rule in globals.css sets `transition-duration: 0.01ms` — scale/shadow jump immediately, no animation | DSGN-04 |

### Project/Image Card Hover Micro-Interaction

| Property | Value | Source |
|----------|-------|--------|
| Image scale | 1.0 → 1.05x on hover | Phase 2 D-09 |
| Image overflow | `overflow-hidden` on card container — image scale clips inside card | Phase 2 D-09 |
| Overlay | Dark gradient fades in: `rgba(0,0,0,0)` → `rgba(0,0,0,0.65)` | discretion |
| Overlay gradient | `linear-gradient(to top, rgba(0,0,0,0.8) 0%, rgba(0,0,0,0.3) 50%, transparent 100%)` | discretion |
| Title/CTA reveal | Project title and CTA link translate from `y: 12px, opacity: 0` → `y: 0, opacity: 1` | discretion |
| Duration | 250ms | discretion — fast enough to feel responsive |
| Easing | `ease-out` | — |
| Implementation | CSS only via Tailwind group-hover utilities (no Motion/GSAP needed) | discretion — simpler, no JS overhead |
| Reduced motion | transitions suppressed by globals.css rule — overlay and title appear instantly | DSGN-04 |

### Hero Parallax

| Property | Value | Source |
|----------|-------|--------|
| Speed differential | Background image moves at 15% of scroll speed (midpoint of 10–20%) | Phase 2 D-12 |
| Implementation | GSAP ScrollTrigger with `scrub: true` on background image `y` property | discretion |
| Background y range | 0px (at top of hero) → `heroHeight * 0.15` px (at hero bottom) | Phase 2 D-12 |
| Desktop only | Applied only when viewport width ≥ 1024px (`lg` breakpoint) | Phase 2 D-15 |
| Mobile fallback | Simple fade-in on page load: opacity 0 → 1, duration 500ms, `ease-out` | Phase 2 D-14 |
| Hero text exit | As user scrolls past hero: text+CTA opacity 1→0 and translateY 0→-30px, tied to ScrollTrigger scrub | Phase 2 D-13 |
| Hero exit distance | Begins when hero bottom crosses 80% viewport, completes at hero bottom at 20% viewport | discretion |
| Scrub | true (scroll-linked, not time-based) — smooth, continuous | Phase 2 D-12 |
| Reduced motion | No ScrollTrigger registered; hero text and image appear at full opacity instantly | DSGN-04 |

## Component Inventory (Primitives to Build)

These are the deliverables of Phase 2. Each is a reusable hook or wrapper component.

| Primitive | Type | Consumes | Used From Phase |
|-----------|------|----------|----------------|
| `useFadeUp` | Hook | GSAP + useGSAP + ScrollTrigger | Phase 3+ |
| `useClipReveal` | Hook | GSAP + useGSAP + ScrollTrigger | Phase 3+ |
| `useStagger` | Hook | GSAP + useGSAP + ScrollTrigger | Phase 3+ |
| `useHeroParallax` | Hook | GSAP + useGSAP + ScrollTrigger | Phase 3 |
| `PageTransition` | Client component | Motion AnimatePresence | app/layout.tsx |
| Card hover styles | Tailwind utility classes (group-hover) | CSS only | Phase 3+ |
| Button hover scale | Tailwind utility classes | CSS only | Button.tsx update |

All hooks must:
- Accept a `ref` or `containerRef` parameter
- Call `gsap.registerPlugin(ScrollTrigger)` inside `useGSAP`
- Respect `prefers-reduced-motion` by checking `window.matchMedia('(prefers-reduced-motion: reduce)').matches` before registering any ScrollTrigger
- Export a cleanup function (GSAP handles via `useGSAP` scope)

## Copywriting Contract

Phase 2 is infrastructure — it delivers no user-facing content, routes, or forms. The only user-visible text in this phase is animation states that reveal already-existing copy (from Phase 3+). No new copywriting is required.

| Element | Copy | Rationale |
|---------|------|-----------|
| Primary CTA | n/a — no CTA in Phase 2 | Infrastructure phase, no new UI |
| Empty state | n/a — no data views in Phase 2 | — |
| Error state | n/a — animation failure silently degrades (reduced motion fallback) | DSGN-04 |
| Destructive confirmation | n/a — no destructive actions | — |

**Animation failure handling:** If GSAP ScrollTrigger fails to initialize (SSR, hydration mismatch), elements must be visible at their final state (full opacity, y: 0) via CSS defaults. Never hide content behind a failed animation.

## Integration Checklist

Executor must verify all of the following after implementation:

- [ ] `useGSAP()` is the only place GSAP animations are registered — no raw `gsap.to()` calls outside the hook
- [ ] `ScrollTrigger.refresh()` is called after Lenis initializes to sync scroll positions
- [ ] All hooks skip ScrollTrigger registration when `prefers-reduced-motion: reduce` is set
- [ ] `AnimatePresence` wraps only `<main>`, with `key={pathname}` — Navbar and Footer are outside it
- [ ] Hero parallax is behind a `useEffect` that checks `window.innerWidth >= 1024` before registering
- [ ] Card hover and button hover are Tailwind CSS only — no Motion `whileHover` on these
- [ ] No custom cursor — standard browser cursor everywhere (Phase 2 D-11)
