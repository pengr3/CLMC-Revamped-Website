**Step 1: Create `components/sections/ProjectsGallery.tsx`**

This is a `'use client'` component. Structure:

```typescript
'use client'

import { useState } from 'react'
import { AnimatePresence, motion } from 'motion/react'
import { ProjectCard } from '@/components/ui/ProjectCard'
import { projects, ALL_CATEGORIES, type ProjectCategory } from '@/data/projects'
import { cn } from '@/lib/utils'
```

State: `const [active, setActive] = useState<ProjectCategory | 'All'>('All')`

Derived: `const filtered = active === 'All' ? projects : projects.filter(p => p.category === active)`

**Filter pills row:** Render `['All', ...ALL_CATEGORIES]` as buttons in a flex-wrap container with `gap-sm mb-3xl`. Each button:
- `onClick={() => setActive(cat)}`
- `key={cat}`
- Classes: `px-md py-sm text-[13px] font-body font-bold uppercase tracking-[0.06em] rounded-full border transition-all duration-150 ease-out`
- Active state (when `active === cat`): `bg-text-primary text-surface-primary border-text-primary`
- Inactive state: `bg-transparent text-text-secondary border-surface-border hover:border-text-secondary hover:text-text-primary`
- Use `cn()` for conditional class merging

**Masonry grid:** CSS columns layout — `columns-1 md:columns-2 lg:columns-3 gap-lg`

Inside the columns div, wrap all cards in `<AnimatePresence mode="popLayout">`. Each card gets a `motion.div` wrapper:
```tsx
<motion.div
  key={project.id}
  layout
  initial={{ opacity: 0, scale: 0.95 }}
  animate={{ opacity: 1, scale: 1 }}
  exit={{ opacity: 0, scale: 0.95 }}
  transition={{ duration: 0.25, ease: 'easeOut' }}
  className="break-inside-avoid mb-lg"
>
  <ProjectCard
    imageSrc={project.imageSrc}
    imageAlt={project.imageAlt}
    title={project.title}
    category={project.category}
    meta={project.meta}
    href={`/projects/${project.id}`}
  />
</motion.div>
```

CRITICAL: Use `project.id` as the key (NOT array index). Use `mode="popLayout"` on AnimatePresence (not default "sync"). Import from `'motion/react'` (NOT `'framer-motion'`).

Note: The `data-fade-up` attribute on the heading div will work with the existing global CSS initial state (`opacity: 0; transform: translateY(40px)`) and GSAP ScrollTrigger that scans for `[data-fade-up]` elements. However, since the gallery page itself doesn't call `useFadeUp`, the heading will remain in its CSS initial state (hidden). To fix this, the `ProjectsGallery` client component should also accept a `sectionRef` and call `useFadeUp` on the parent section.

**Revised approach:** Make the entire page a client component wrapper OR make ProjectsGallery responsible for the full section including heading. Choose the simpler option: make `app/projects/page.tsx` a simple Server Component that renders `<ProjectsGallery />` and move the heading + description INSIDE `ProjectsGallery.tsx` (since it's already a client component with access to hooks). Add `useRef` + `useFadeUp` inside `ProjectsGallery` wrapping the heading div.

Final structure of `ProjectsGallery.tsx`:
- `const sectionRef = useRef<HTMLDivElement>(null)`
- `useFadeUp(sectionRef)` — animates the `data-fade-up` heading
- Return `<div ref={sectionRef}>` containing heading (`data-fade-up`), filter pills, and masonry grid
